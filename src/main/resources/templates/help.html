General

Welcome to KWIX, the fast and easy employee and payroll management system. KWIX was concieved to store employee information, record time worked, and to record which employees worked at which jobsite.

Quick Overview:

Main pages:

  • Dashboard: Home page
  • Employees: Create and Manage Employees
  • Payroll: Create and Manage Paystubs for Employees
  • Jobsites: Create and Manage Jobsites
Each main page has a "back" arrow that will take you back to the previous page. Additionally, each page has a recently deleted link that will show inactive records. "Inactive" records are records that have been "deleted". When a record is "deleted" it is only set as "inactive". This is to prevent accidental deletions. To permanently delete a record, you must click "delete" on the recently deleted page.

The Dashboard will display recent updates and useful information. There is a link to the Help page (this page) for troubleshooting. There is also a contact link that links to the developer email.

Login/Logout/Session:

When logging, in your "session" will be stored in your browser for a brief period of time. Your "session" will not store any drafts or incompleted entries for Employees, Payroll, or Jobsites. You must submit all entries before logging out if you want to save your progress. Additionally, "sessions" are designed to expire after a period of time. After a session expires you will be required to log in again. It is encouraged that if your computer is around others or your computer is shared that you logout after you are done using KWIX.

Employees

Manage Employees

When navigating to the "Employees" page an authorized user can do the following:

  • View all employees
  • Add new employees
  • Make changes to existing Employees
  • Remove inactive employees

Add Employee

To add a new employee, simply fill out the form and click 'Add Employee'. This will record the values entered into the form and create a new employee and save that employee to the KWIX employee database. Be sure to enter all values correctly.

A First Name and a Last Name must be entered into their respective fields. First Name and Last Name must only contain alphabetic characters. An error will appear if numeric values (0-9) are entered.

A Phone number must be provided to create a unique ID for the employee. ID's are important as this allows KWIX to find employees quickly. Providing a Phone number also makes it easy to contact an individual employee easily.

Pay must be entered. Pay is an employees hourly wage (if jobsite is 'Contract' then the hourly pay will be overwritten, more on this in the Jobsites section). This can be updated along with each of the fields mentioned previously.

Edit Employee

To edit an Employee's information, just click the "pencil" icon. This will take the user to a page for specifically editing data related to the employee. Each of the fields

  • First Name
  • Last Name
  • Phone
  • Pay

To update an employee's information you can change any of these fields and when the update button is clicked then you will be returned to the "Manage Employees" page with the updated employee information. Each of the fields have the same constraints as the Manage Employees page. An error message will appear if a value was entered incorrectly, and the employee will not be updated.

If the fields are edited and the Reset button is clicked then each of the fields will be Reset to how they originally were when the employee was created or last updated. You can also click the "<-" next to the employee's name to return to the previous page.

To delete an employee, simply click the delete button and the employee will be set to 'inactive'. When the employee is set to "inactive" they will be moved to the recently deleted page.

Recently Deleted Employees Page

Clicking the recently deleted link will take the user to the recently deleted page. Here you have two options. Restore or Delete. Clicking Restore will set the employee back to "active" and then they can be seen again on the "Manage Employees" page. Clicking Delete will PERMANENTLY delete the employee from the database. This is a permanent action and the employee will be gone forever.

As you get familiar with KWIX and all of the pages you'll begin to notice that they are all quite similar.

Payroll

Manage Payroll

When navigating to the "payroll" page an authorized user can do the following:

  • View all employees
  • Add new paystubs
  • Make changes to existing paystubs
  • Remove inactive paystubs
  • Export paystub data as CSV file.

What is a paystub? A paystub is a record for the hours worked and gross pay for a given day of the week. Each paystub has its unique ID.

Add Paystub

The "Manage Payroll" page shows all "active" paystubs as well as all "active" employees. To view "deleted" or "inactive" paystubs click the Recently Deleted link.

To record hours worked by an employee, click the "+" sign in the Add Hours column in the Employees: table. If you would like to to view employee info and make changes you can click the "employee id" link and you will be redirected to the "edit employee" page for the selected employee.

Clicking the "+" sign will redirect you to the Manage Paystubs For [Employee] employee page. Here you can see all recorded paystubs for an employee. Additionally their Total Hours worked and Total Gross Pay (pay before tax) will be calculated and shown.

To record hours worked the following fields must be filled out:

  • Select Jobsite
  • Horly Pay
  • Hours Worked
  • Day

To create a paystub, a Jobsite must be selected, Hourly Pay must be entered, Hours Worked must be entered, and Day worked must be entered as "mm/dd/yyyy" i.e 01/01/2000. DO NOT DEVIATE FROM THIS. Once all fields have been entered correctly, clicking the Add button will create a paystub for said employee.

Clicking the Reset button will reset the page and cancel the new record. If a field contains "junk" data or a bad input an error will be displayed.

You'll notice that Employee ID, Full Name, Hourly Pay fields are already filled with data. This is data brought in from the employee's record. In other words it is data that was entered when the employee was created.

Employee ID, Full Name, Hourly Pay fields are read-only, they cannot be edited, with the exception of Hourly Pay. Hourly Pay is set when the employee is added to the database. This value can be changed when creating a paystub because a jobsite may create conditions in which additional pay is awarded, in otherwords this page was designed to give Hourly Pay flexibility when recording pay and time worked.

Edit Paystub

To edit a paystub click the "pencil" icon in the edit column. This can be done from the Manage Payroll page and the Manage Paystubs For employee page. Regardless of which page, clicking the "pencil" icon will redirect you to the edit page for the selected paystub. This is the Paystub For [Day], [Date] | [Employee] page. The following fields can be edited:

  • Select Jobsite
  • Pay
  • Hours Worked
  • Day
To save changes, click the Update button. Clicking the Reset button will clear all edited fields and return them as they were. Clicking the Delete button will send the current pay stub to Recently Deleted page.

If you wish to view the employee that "owns" this paystub refer to the table at the bottom of the page and click the Employee link. Additionally, the saved Jobsite, Pay, Hours Worked, and Gross Pay for the selected paystub will be shown.

Recently Deleted Paystubs Page

Clicking the Recently Deleted link will take the user to the Recently Deleted Paystubs page. Here you have two options. Restore or Delete. Clicking Restore will set the employee back to "active" and then they can be seen again on the "Manage Payroll" page.

Clicking Delete will PERMANENTLY delete the paystub from the database. This is a permanent action and the paystub will be gone forever, but the employee will remain untouched. Only the paystub will be deleted forever.

Export

Clicking the Export button will retrieve all paystub records from the payroll database, organize all records into rows, and download all data as a (dot)CSV file. (dot)CSV files are compatible with all spreadsheet software (google sheets, excel, etc). Using your spreadsheet software of choice, you can then save the CSV file as whichever file you please.

Open the CSV file in your spreadsheet software, then click file -> save as -> give the file a name -> click save.

Jobsites

Manage Jobsites:

When navigating to Jobsites you can do the following:

  • Create New Jobsites
  • View Existing Jobsites
  • Modify Existing Jobsites
  • Permanently Remove Inactive Jobsites

Add Jobsite

To create a new jobsite data must be entered into the following fields:

  • Jobsite Name
  • Contract Job?
A jobsite name must be provided, usually it is the name of the street the jobsite is located on. It must be declared if the jobsite is a Contract Job or not. Contract jobs have a fixed pay and will override an employees hourly wage when creating a paystub associated with that job.

Once these fields have been filled out correctly, to add the jobsite to the KWIX database click Add Jobsite. Clicking Reset will clear all fields and restore the form as it was.

Edit Jobsites

To edit a jobsite, click the "pencil" icon. You will be redirected to the Manage [Jobsite Name] page. Here you can edit the following:

  • Jobsite Name
  • Contract Job
To save your changes click Update button. To delete this jobsite click Delete button and the jobsite will be set to inactive. To cancel all changes click Reset and everything will be reset to how it was.

Recently Deleted Jobsite

Clicking Recently Deleted on the jobsites page will redirect you to the Recently Deleted Jobsites page that shows all "inactive" jobsites. To PERMANENTLY delete a jobsite, click the delete button. Doing this is will remove the jobsite from the database forever. To restore the jobsite and set it back to "active", click the restore button. If restored, the jobsite will be visible again on the Manage Jobsite page.

Search

Searching for Records:

When navigating to the search page you can do the following:

  • Select tables to include in your search.
  • Select columns that you would like to include in your search.
  • View Employees, Paystubs, and Jobsites returned in your search.

Functionality:

The search page was added to include the ability to create robust and complex searches.

For Example:

  • Search for all paystubs related to a specific job, or search for all employees assigned to a specific job.
  • Search for records using a range of dates. i.e. from 08/15/2024 to 12/20/2024

These are just a few examples. The search page will be a powerful tool for managing records. A nice feature, is that you can search for deleted records, so long as they have not been permanetly deleted. This can be accomplished by using the status button and selecting "active" or "deleted".

To use a table in the search simply check the tables you want to search in. You can select one or many.

    Tip: use all of the tables in combination for robust search results.

Completing a search:

  • Select tables using the checkboxes
  • Select the columns you want to bring in
  • Type in the value you want to search by or use the "any" button
  • Using "any" will bring in all values related to that column
  • Click search and the result will match the inputs you selected

Preventing Problems:

Once you select a table and select a column you must enter a value or use the "any" button. You cannot leave a field blank once you have selected it. If you decided you don't want to search by that field, just click the column button again and it will go away.

"Status" and "Contract?"

These fields are special because they do not take text values. You must select "yes" or "no". These are radio buttons. Radio buttons only allow you to select one or the other. Once you select a radio button you cannot unselect it. If you want to "unselect" a radio button you must click the "reset" button below. Keep in mind clicking "reset" will reset the entire page.
